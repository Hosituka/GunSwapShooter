# 命令
- 件名：PointObjectGeneraterにObjectPoolを実装しようとしたら、まともに動作しなくなった問題を、私の代わりに見つけてほしい。
- 詳細：モバイルブラウザで動くゲームである都合上、スパイク削減のため、ObjectPoolを実装しようとしたら、PointObjectGeneraterの生成の同期が上手くいかなくなりました。これはゲームの中核となる部分であり、色々なクラスが複合的に関わっていて、私の理解を超えてきたため、一回あなたに生成の同期が崩れる原因を調べてほしいです。コードの修正は自分で行いますので、あなたはどうコードを修正すれば良いのかを伝えてください。各クラスorインスタンスの役割は以下の通りです。
- 調べるにあたって：現在ステージ1に関連するものだけをObjectPoolしようとしているため、ステージ2に当たるファイルは調査しなくてよいです。
# 各クラスのインスタンスの役割(ステージ1に関連する)
- PointObjectGeneraterインスタンス：各ステージごとにあるPointObjectGeneraterクラスの処理をまとめる抽象クラスです。
- PointObjectGenerater1と2：ステージ1と2に対応するpointobjectGeneraterの具象クラスです。switch文による型チェックの内容や、保持するObjectPoolの種類が違います。現在はステージ1に対応するPointObjectGenerater1だけを見てください
- TimeKeeper：ある時点においてPointObjectが一つor複数生成された時、それらのライフサイクルをまとめるものです。また、PointObjectGeneraterに次のPointObjectの生成の仕方を伝えることも行ってます。
- PointObject：ARシューティングゲームに登場する各的の共通する振る舞いや状態が書かれている抽象クラス、今回の命令においてはステージ1に登場する具象クラスだけを見てください　つまりMoveBlueTargetは見なくてよいです。
- RedTarget：ステージ1に出てくる赤い弾に反応する的
- BlueTarget：ステージ1に出てくる青い弾に反応する的
- ButtonMashingTarget：ステージ1に出てくる連打することで撃墜判定となる的
- BlueRedTarget：ステージ1に出てくる左半分が青、右半分が赤な同時押し的
- RedBlueTarget：ステージ1に出てくる左半分が赤、右半分が青な同時押し的
